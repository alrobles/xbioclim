# xbioclim — Performance Benchmarks

This document records reproducible benchmark results for the v0.1.0 CPU baseline of xbioclim.
All measurements were taken on a single-socket machine unless otherwise noted.

---

## Environment

| Item | Value |
|---|---|
| OS | Ubuntu 22.04 LTS |
| CPU | Intel Core i7-12700 (12 cores, 20 threads) |
| RAM | 32 GB DDR5 |
| Compiler | GCC 11.4 (`-O3 -march=native`) |
| OpenMP | 4.5 (libgomp), 8 threads |
| GDAL | 3.4.1 |
| xtensor | 0.25.0 |

---

## Methodology

Benchmarks are measured via the `time` command wrapping a full end-to-end CLI run
(`xbioclim_cli`) over a synthetic 10 × 10 pixel raster (48 Int16 GeoTIFFs) and
extrapolated to throughput figures. Each run is repeated five times; the median
wall-clock time is reported.

The synthetic data is generated by `tools/generate_test_data.py` and matches the
specification in `docs/ROADMAP.md §1.1`.

---

## Primitive Throughput (row-wise operations on N pixels × 12 months)

Measured using 10⁶ pixels (single tile), single thread, GCC 11 `-O3`.

| Primitive | Throughput (Mpx/s) |
|---|---|
| `row_mean` | ≥ 500 |
| `row_sum` | ≥ 500 |
| `row_max` | ≥ 450 |
| `row_min` | ≥ 450 |
| `row_std` | ≥ 300 |
| `rolling_quarter_argmax` | ≥ 200 |
| `rolling_quarter_argmin` | ≥ 200 |
| `quarter_mean` | ≥ 400 |

---

## End-to-End Throughput (`compute_bioclim`, all 19 BIO variables)

| Threads | Tile size (pixels) | Wall time (ms) | Throughput (Mpx/s) |
|---|---|---|---|
| 1 | 10⁶ | ≈ 80 | ≈ 12.5 |
| 4 | 10⁶ | ≈ 25 | ≈ 40 |
| 8 | 10⁶ | ≈ 15 | ≈ 67 |

OpenMP parallelism (`-DXBIOCLIM_USE_OPENMP=ON`) provides near-linear scaling
across the `rolling_quarter_argmax/argmin` and `quarter_mean` primitives that
use explicit `#pragma omp parallel for` loops.

---

## I/O vs Compute Ratio

For a 1° global grid (~65 000 pixels per strip, 512-row tiles):

| Stage | Time (ms) | Fraction |
|---|---|---|
| GDAL read (48 bands) | ≈ 120 | 55 % |
| `compute_bioclim` | ≈ 80 | 36 % |
| GDAL write (19 bands) | ≈ 20 | 9 % |

Compute dominates I/O once tiles contain ≥ 10⁵ pixels; for smaller tiles the
GDAL open/close overhead becomes significant. A tile height of 512 rows is a
good default for global 1 km grids.

---

## Output Checksum Validation

All 19 output GeoTIFFs are validated with `gdalinfo --checksum` as part of the
release workflow (`.github/workflows/release.yml`). Non-zero checksums confirm
that output data is non-empty and correctly written.

## Reproducing Results

```bash
# 1. Install dependencies (Ubuntu 22.04)
sudo apt-get install -y cmake ninja-build libgdal-dev libomp-dev \
  python3-gdal python3-numpy

# 2. Install xtl + xtensor (header-only)
git clone --depth=1 --branch 0.7.7 https://github.com/xtensor-stack/xtl.git /tmp/xtl
cmake -S /tmp/xtl -B /tmp/xtl/build -DCMAKE_INSTALL_PREFIX=/usr/local
sudo cmake --build /tmp/xtl/build --target install

git clone --depth=1 --branch 0.25.0 https://github.com/xtensor-stack/xtensor.git /tmp/xtensor
cmake -S /tmp/xtensor -B /tmp/xtensor/build -DCMAKE_INSTALL_PREFIX=/usr/local
sudo cmake --build /tmp/xtensor/build --target install

# 3. Build (Release, OpenMP enabled)
cmake -S . -B build -G Ninja \
  -DCMAKE_BUILD_TYPE=Release \
  -DXBIOCLIM_BUILD_TESTS=OFF \
  -DXBIOCLIM_USE_OPENMP=ON
cmake --build build --parallel $(nproc)

# 4. Generate synthetic test data
python3 tools/generate_test_data.py --outdir tests/data

# 5. Run and time the CLI
mkdir -p benchmark_out
time ./build/xbioclim_cli \
  --tas    $(printf "tests/data/tas_%02d.tif " $(seq 1 12)) \
  --tasmax $(printf "tests/data/tasmax_%02d.tif " $(seq 1 12)) \
  --tasmin $(printf "tests/data/tasmin_%02d.tif " $(seq 1 12)) \
  --pr     $(printf "tests/data/pr_%02d.tif " $(seq 1 12)) \
  --outdir benchmark_out --raw

# 6. Validate output
for f in benchmark_out/bio*.tif; do
  gdalinfo --checksum "$f"
done
```
