# rxbioclim — R Wrapper Roadmap

> **Goal:** Expose the `xbioclim` C++17 bioclimatic engine to R users through a thin Rcpp Module binding, following the same architectural patterns used by the [`terra`](https://github.com/rspatial/terra) package. The R package requires **zero C++ knowledge** from the end user — just file paths in, `SpatRaster` objects (or named lists) out.

## 1. Background & Motivation

The `xbioclim` C++ library computes all 19 CHELSA/WorldClim bioclimatic variables (BIO01–BIO19) at high performance using xtensor, GDAL block I/O, OpenMP parallelism, and optional CUDA GPU acceleration. However, the primary user base — ecologists, macroecologists, and conservation biologists — works almost exclusively in R.

The solution is a **thin R wrapper package** (`rxbioclim`) that:

- Exposes a single user-facing function `compute_bioclim()`.
- Accepts file path vectors or glob patterns for the 48 input GeoTIFFs.
- Returns results as named `SpatRaster` objects (via `terra`) or a named list of output file paths.
- Handles **all C++ complexity invisibly**: GDAL I/O, tile processing, memory management, NaN propagation.

### Design Philosophy (from `terra` analysis)

The [`terra` architecture analysis](terra_architecture_analysis.md) establishes the gold standard for C++/R geospatial bindings:

| terra Pattern | rxbioclim Application |
|---|---|
| Single `RCPP_MODULE(spat)` | Single `RCPP_MODULE(xbioclim)` |
| S4 opaque handle (`pntr = "C++Object"`) | S3/R6 handle wrapping `Rcpp_XBioclimEngine` |
| Two-tier dispatch (C++ fast path + R fallback) | All computation in C++; R only validates paths and formats output |
| Message-based error propagation (`SpatMessages`) | Return status codes; R checks and throws `stop()` |
| Options object (`SpatOptions`) | `BioclimOptions` list or S3 object |

## Phase R1 — Rcpp Module Binding

### R1.1 The Module File: `src/RcppModule.cpp`

This is the **only new C++ file** the R package adds on top of the vendored xbioclim sources.

```cpp
// src/RcppModule.cpp
#include <Rcpp.h>
#include "xbioclim/gdal_io.hpp"
#include "xbioclim/bioclim.hpp"

using namespace Rcpp;
using namespace xbioclim;

// ---------------------------------------------------------------------------
// Helper: convert a std::vector<std::string> of file paths into a GdalReader
// ---------------------------------------------------------------------------
static GdalReader make_reader(StringVector tas_files,
                               StringVector tasmax_files,
                               StringVector tasmin_files,
                               StringVector pr_files) {
    std::vector<std::string> tas(tas_files.begin(),     tas_files.end());
    std::vector<std::string> tasmax(tasmax_files.begin(), tasmax_files.end());
    std::vector<std::string> tasmin(tasmin_files.begin(), tasmin_files.end());
    std::vector<std::string> pr(pr_files.begin(),       pr_files.end());
    return GdalReader(tas, tasmax, tasmin, pr);
}

// ---------------------------------------------------------------------------
// Standalone exported function — the core of the R API
// ---------------------------------------------------------------------------

// [[Rcpp::export]]
List compute_bioclim_r(StringVector tas_files,
                       StringVector tasmax_files,
                       StringVector tasmin_files,
                       StringVector pr_files,
                       std::string  outdir,
                       int          tile_size = 512,
                       int          threads   = -1) {
    try {
        GdalReader reader = make_reader(tas_files, tasmax_files,
                                        tasmin_files, pr_files);
        GdalWriter writer(outdir, reader.metadata());

        reader.process_tiles(tile_size, threads,
            [&](const ClimateBlock& block, int tx, int ty) {
                BioBlock bio = compute_bioclim(block);
                writer.write_block(tx, ty, bio);
            });

        writer.finalize();

        // Return named list of output file paths: list(bio01="outdir/bio01.tif", ...)
        List result;
        for (int i = 1; i <= 19; ++i) {
            char key[8];
            std::snprintf(key, sizeof(key), "bio%02d", i);
            result[key] = writer.output_path(i);
        }
        return result;

    } catch (const std::exception& e) {
        Rcpp::stop(e.what());
    }
}

// ---------------------------------------------------------------------------
// (Optional) Rcpp Module for stateful engine — advanced users / benchmarking
// ---------------------------------------------------------------------------

RCPP_MODULE(xbioclim) {
    using namespace Rcpp;
    // Expose the version string
    function("xbioclim_version", []() -> std::string {
        return XBIOCLIM_VERSION_STRING;
    });
}
```

> **Note:** For the simple one-function use case, `// [[Rcpp::export]]` on `compute_bioclim_r()` is sufficient. The `RCPP_MODULE` block is reserved for future stateful engine exposure (e.g., streaming large datasets in chunks from R).

### R1.2 Type Bridging Summary

| xbioclim C++ type | Rcpp bridge | R type |
|---|---|---|
| `std::vector<std::string>` | `Rcpp::StringVector` | `character` vector |
| `std::string` | `std::string` (auto) | `character(1)` |
| `int` | `int` (auto) | `integer(1)` |
| `ClimateBlock` (internal) | not exposed | — |
| `BioBlock` (internal) | not exposed | — |
| Output file paths | `Rcpp::List` of strings | named `list` |

## Phase R2 — R-Side Wrapper Functions

### R2.1 `R/bioclim.R` — The User-Facing Function

```r
#' Compute 19 Bioclimatic Variables from Monthly Climate Rasters
#'
#' Wraps the xbioclim C++ engine to compute BIO01–BIO19 following the
#' CHELSA/WorldClim definitions. Input files must be GeoTIFFs with integer
#' raw values and matching spatial extent/resolution.
#'
#' @param tas     Character vector of 12 file paths for monthly mean temperature
#'                (ordered Jan–Dec), or a glob pattern like \code{"data/tas_??.tif"}.
#' @param tasmax  Character vector of 12 paths for monthly maximum temperature.
#' @param tasmin  Character vector of 12 paths for monthly minimum temperature.
#' @param pr      Character vector of 12 paths for monthly precipitation.
#' @param outdir  Output directory for the 19 BIO GeoTIFFs. Created if absent.
#' @param options A \code{BioclimOptions} object (see \code{\link{BioclimOptions}}).
#' @param as_spatraster Logical. If \code{TRUE} (default) and \code{terra} is
#'                installed, return a \code{SpatRaster} stack; otherwise return
#'                a named list of file paths.
#'
#' @return A \code{SpatRaster} with 19 layers (BIO01–BIO19), or a named list
#'         of output file paths if \code{as_spatraster = FALSE}.
#'
#' @examples
#' \dontrun{{
#' bio <- compute_bioclim(
#'   tas    = Sys.glob("chelsa/tas_??.tif"),
#'   tasmax = Sys.glob("chelsa/tasmax_??.tif"),
#'   tasmin = Sys.glob("chelsa/tasmin_??.tif"),
#'   pr     = Sys.glob("chelsa/pr_??.tif"),
#'   outdir = "output/bioclim"
#' )
#' terra::plot(bio[["bio01"]])
#' }}
#' @export
compute_bioclim <- function(tas, tasmax, tasmin, pr,
                             outdir = ".",
                             options = BioclimOptions(),
                             as_spatraster = TRUE) {
  # --- Expand globs if single-string patterns provided ---
  tas    <- .expand_paths(tas)
  tasmax <- .expand_paths(tasmax)
  tasmin <- .expand_paths(tasmin)
  pr     <- .expand_paths(pr)

  # --- Validate ---
  .check_12_files(tas,    "tas")
  .check_12_files(tasmax, "tasmax")
  .check_12_files(tasmin, "tasmin")
  .check_12_files(pr,     "pr")

  if (!dir.exists(outdir)) dir.create(outdir, recursive = TRUE)

  # --- Call C++ engine ---
  result <- compute_bioclim_r(
    tas_files    = tas,
    tasmax_files = tasmax,
    tasmin_files = tasmin,
    pr_files     = pr,
    outdir       = outdir,
    tile_size    = options$tile_size,
    threads      = options$threads
  )

  # --- Format output ---
  if (as_spatraster && requireNamespace("terra", quietly = TRUE)) {
    paths <- unlist(result)
    r <- terra::rast(paths)
    names(r) <- names(result)
    return(r)
  }

  result
}

# Internal helpers
.expand_paths <- function(x) {
  if (length(x) == 1 && grepl("[*?]", x)) Sys.glob(x) else x
}

.check_12_files <- function(paths, varname) {
  if (length(paths) != 12)
    stop(sprintf(
      "'%s' must resolve to exactly 12 files (got %%d).",
      varname, length(paths)
    ))
  missing <- paths[!file.exists(paths)]
  if (length(missing) > 0)
    stop(sprintf("Files not found for '%s':\n  %%s",
                 varname, paste(missing, collapse = "\n  ")))
}
```

### R2.2 `R/options.R` — `BioclimOptions`

Mirrors `terra`'s `SpatOptions` pattern (a simple list-backed constructor):

```r
#' Create a BioclimOptions Object
#'
#' Controls processing parameters for \code{\link{compute_bioclim}}.
#'
#' @param tile_size Integer. Tile width/height in pixels (default 512).
#' @param threads   Integer. Number of OpenMP threads; \code{-1} uses all cores.
#' @param nodata    Numeric. NoData sentinel value in input files (default -9999).
#'
#' @return A named list of class \code{"BioclimOptions"}.
#' @export
BioclimOptions <- function(tile_size = 512L,
                            threads   = -1L,
                            nodata    = -9999) {
  structure(
    list(tile_size = as.integer(tile_size),
         threads   = as.integer(threads),
         nodata    = as.numeric(nodata)),
    class = "BioclimOptions"
  )
}

#' @export
print.BioclimOptions <- function(x, ...) {
  cat("BioclimOptions:\n")
  cat("  tile_size:", x$tile_size, "\n")
  cat("  threads:  ", x$threads,   "\n")
  cat("  nodata:   ", x$nodata,    "\n")
  invisible(x)
}
```

## Phase R3 — GDAL Detection & Build System

### R3.1 `configure` Script

R's build system runs `configure` before compiling. This script detects GDAL using `gdal-config` (the standard GDAL CLI tool):

```bash
#!/bin/sh
# configure — auto-generated; detects GDAL for rxbioclim

GDAL_CONFIG=$(command -v gdal-config 2>/dev/null)

if [ -z "$GDAL_CONFIG" ]; then
  echo "ERROR: gdal-config not found. Please install GDAL >= 3.0." >&2
  echo "  Ubuntu/Debian:  sudo apt-get install libgdal-dev" >&2
  echo "  macOS (Homebrew): brew install gdal" >&2
  exit 1
fi

GDAL_VERSION=$($GDAL_CONFIG --version)
echo "Found GDAL $GDAL_VERSION at $GDAL_CONFIG"

PKG_CPPFLAGS="$($GDAL_CONFIG --cflags) -std=c++17 -I../src/include"
PKG_LIBS="$($GDAL_CONFIG --libs)"

# Write Makevars
sed \
  -e "s|@PKG_CPPFLAGS@|${PKG_CPPFLAGS}|" \
  -e "s|@PKG_LIBS@|${PKG_LIBS}|" \
  src/Makevars.in > src/Makevars

echo "Makevars written."
```

### R3.2 `src/Makevars.in`

```makefile
PKG_CPPFLAGS = @PKG_CPPFLAGS@
PKG_LIBS     = @PKG_LIBS@

# Enable C++17
CXX_STD = CXX17
```

### R3.3 Vendoring xtensor

Since xtensor is header-only, copy `xtensor/include/xtensor/` into `src/include/xtensor/` and add `-I$(CURDIR)/include` to `PKG_CPPFLAGS`. This avoids any system-level dependency on xtensor for R package users.

### R4.1 `tests/testthat/test-bioclim.R`

Uses the same analytical fixture values from the C++ test suite (ROADMAP §1.3):

```r
library(testthat)
library(rxbioclim)

# Paths to the synthetic 10×10 GeoTIFFs generated by xbioclim test suite
data_dir <- system.file("testdata", package = "rxbioclim")

tas_files    <- file.path(data_dir, sprintf("tas_%02d.tif",    1:12))
tasmax_files <- file.path(data_dir, sprintf("tasmax_%02d.tif", 1:12))
tasmin_files <- file.path(data_dir, sprintf("tasmin_%02d.tif", 1:12))
pr_files     <- file.path(data_dir, sprintf("pr_%02d.tif",     1:12))

test_that("compute_bioclim returns 19 output paths", {
  result <- compute_bioclim(
    tas = tas_files, tasmax = tasmax_files,
    tasmin = tasmin_files, pr = pr_files,
    outdir = tempdir(), as_spatraster = FALSE
  )
  expect_length(result, 19)
  expect_named(result, sprintf("bio%02d", 1:19))
})

test_that("BIO01 mean annual temperature equals 6.5", {
  skip_if_not_installed("terra")
  bio <- compute_bioclim(
    tas = tas_files, tasmax = tasmax_files,
    tasmin = tasmin_files, pr = pr_files,
    outdir = tempdir()
  )
  val <- mean(terra::values(bio[["bio01"]]), na.rm = TRUE)
  expect_equal(val, 6.5, tolerance = 1e-3)
})

test_that("BIO07 annual temperature range equals 13.0", {
  skip_if_not_installed("terra")
  bio <- compute_bioclim(
    tas = tas_files, tasmax = tasmax_files,
    tasmin = tasmin_files, pr = pr_files,
    outdir = tempdir()
  )
  val <- mean(terra::values(bio[["bio07"]]), na.rm = TRUE)
  expect_equal(val, 13.0, tolerance = 1e-3)
})

test_that("wrong number of files raises error", {
  expect_error(
    compute_bioclim(
      tas = tas_files[1:11], tasmax = tasmax_files,
      tasmin = tasmin_files, pr = pr_files,
      outdir = tempdir()
    ),
    regexp = "'tas' must resolve to exactly 12 files"
  )
})
```
